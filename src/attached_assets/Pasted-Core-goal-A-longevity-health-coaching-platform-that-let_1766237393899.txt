Core goal

A longevity health coaching platform that:

lets users upload/enter lab results (biomarkers),

normalizes & maps different lab formats into a standard schema,

runs a rule-based decision engine + AI to generate personalized lifestyle + lab follow-up guidance,

tracks progress over time with dashboards,

supports paid subscriptions via Stripe,

includes strict medical safety guardrails, audit logs, and privacy-first architecture.

IMPORTANT SAFETY RULES (must be enforced)

The app must NOT provide medical diagnosis or treatment.

The app must NOT provide specific medication/TRT/peptide dosing instructions.

If user asks about dosing, the assistant should refuse and recommend consulting a licensed clinician.

The app should provide educational, non-prescriptive guidance: lifestyle, testing cadence, questions to ask a clinician, risk flags, and general informational summaries.

Add mandatory Medical Disclaimer acceptance during onboarding and before viewing any personalized guidance.

Tech stack

Frontend: Next.js (App Router) + TypeScript + Tailwind CSS

Backend: Next.js API routes (or Express if needed)

Database: Supabase Postgres (preferred) with Row Level Security

Auth: Supabase Auth (email/password + magic link)

Payments: Stripe subscriptions (monthly + yearly)

AI: OpenAI API (server-side only). Use environment variables for keys.

Charts: simple chart library for biomarker trends

Decision engine (moat)

Implement a Rules Engine that triggers flags and guidance based on biomarkers + questionnaire.
Examples (keep general, non-medical):

If hematocrit is above a threshold → show “Potential risk flag” and advise discussing with clinician; suggest hydration, re-check timing, possible secondary causes; emphasize not medical advice.

If LDL/ApoB high → lifestyle focus: diet quality, fiber, activity; recommend discussing advanced lipid panel options with clinician.

If A1c elevated → lifestyle focus: weight, movement, sleep; recommend follow-up testing cadence.

Rules engine must produce:

flags: {severity: low/medium/high, message, suggested_next_steps}

follow_up_labs: list of labs to consider discussing with clinician, with timing suggestions (e.g., “in ~8–12 weeks”)

lifestyle_priorities: ranked list

Then pass this structured output into AI to produce a personalized, motivational plan in a confident but non-prescriptive tone.

Audit log & safety

Create an audit_logs table recording:

timestamp

Create a “Report an issue” button on recommendations that stores user feedback.

Add a “Red Flags” guard:

If user mentions severe symptoms (e.g., chest pain, fainting, suicidal ideation, acute shortness of breath) → show emergency message: “Seek immediate medical help / local emergency number” and do not generate advice.
